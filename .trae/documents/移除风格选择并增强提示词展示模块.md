此计划旨在移除风格选择功能，并在生成流程中新增一个清晰的提示词展示模块。

### 1. 前端改造 (`GeneratorApp.tsx`)

*   **移除风格选择器**:
    *   删除 `STYLES` 常量。
    *   删除 `selectedStyle` 状态。
    *   删除 UI 中的“选择一种风格”区域（Cards Grid）。
    *   移除 API 调用 (`/api/analyze` 和 `/api/generate`) 中的 `style` 参数。
*   **新增提示词展示模块**:
    *   设计一个新的状态 `analysisError` 用于捕获和展示错误信息。
    *   在“一键生成”按钮下方，添加一个专门的提示词展示区域。
    *   **展示逻辑**:
        *   **Loading 态**: 当 `isAnalyzing` 为 true 时，显示“AI 正在思考最佳构图...”的骨架屏或 Loading 动画。
        *   **Success 态**: 当 `generatedPrompt` 存在时，显示完整的英文提示词，应用 `whitespace-pre-wrap` 样式确保自动换行，并使用等宽字体或引用块样式增加可读性。
        *   **Error 态**: 当 `analysisError` 存在时，显示红色错误提示框。

### 2. 后端 API 适配

*   **更新 `/api/analyze`**:
    *   移除对 `style` 参数的接收。
    *   修改 Gemini System Prompt：不再依赖 `specificInstruction`，而是将“小红书风格”的核心要素（明亮光线、生活化、精致构图、低饱和度色彩等）硬编码在 System Prompt 中，指示 AI 根据用户输入自动适配最佳视觉风格。
*   **更新 `/api/generate`**:
    *   移除 `style` 参数的接收。
    *   在 Mock 模式下，移除对 `style` 的判断，统一使用高质量的生活化/Ins 风占位图逻辑。
    *   在存入数据库时，`style` 字段可存为默认值 "Auto" 或空字符串。

### 3. 数据库适配

*   `prisma/schema.prisma` 中的 `style` 字段是可选的 (`String?`)，因此无需修改 Schema，只需在写入时传入默认值即可。

### 4. 验证计划

*   验证界面不再显示风格选择。
*   输入中文文本，点击生成，确认能够看到“AI 正在思考...”的加载状态。
*   确认生成的英文提示词能够完整展示并自动换行。
*   确认图像生成依然正常工作，且风格保持“小红书”味（由 Prompt 控制）。
